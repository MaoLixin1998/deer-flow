## 3. 已固化页面状态

本次固化包含 5 类状态：

```text
1. 工作台默认态：会话列表 + Agent 对话 + 素材面板。
2. 画板编辑态：默认选中第一个画板工具，右侧工具抽屉默认打开。
3. 左侧收起态：左侧合并为单列入口，不出现双竖条。
4. 规划帽子展开态：竖排待办列表，完成打勾，未完成空框。
5. 设置页：Runtime、降级状态、Mock 工具调用登记。
```

## 4. 固化规则

后续开发必须遵守：

```text
1. 页面必须全中文。
2. 白灰极简风格继续保留。
3. 可以参考 Codex 的交互气质，但不能照抄 Codex 的功能文案。
4. 左右面板都可以拖拽收起。
5. 右侧画板从右侧推入，不能突然跳出。
6. 画板默认选择第一个工具，并默认打开工具抽屉。
7. 对话输入框按钮放在框内。
8. 规划帽子默认只展示当前阶段，展开后是竖排事项列表。
9. 不允许恢复多余重边框、双竖条、过大留白和无作用按钮。
```

## 5. 后续使用方式

代码重构时按以下顺序对照：

```text
1. 先打开当前运行中的前端页面。
2. 再改 frontend/apps/web/。
3. 改完后和当前前端页面效果对照，确认没有回退。
4. Figma 页面：03 Phase 1.7 当前前端界面固化，只作为辅助回看。
5. 如果交互规则改变，必须先更新 Phase 文档，再视情况更新 Figma 记录。
```

## 6. 与 Phase 1.7 的关系

Phase 1.7 的目标是前端静态工程化整理。

本次 Figma 固化的作用：

```text
1. 防止拆组件时视觉回退。
2. 防止左右面板交互被误改。
3. 防止画板默认态被改回局部重绘。
4. 防止规划帽子被改回横排卡片。
```
